## 1. 概要

本アプリケーションは、ブラウザ上で音声ファイルまたは動画ファイルを読み込み、音声解析に応じてリアルタイムにビジュアル表現を行う **オーディオアナライザー / オーディオビジュアライザー** である。

本アプリは以下の2つの利用目的を持つ。

1. **再生モード**
   - 音声や動画を再生しながら、リアルタイムなアナライザー表現を楽しむ
   - 複数スロットを再生キューとして扱う

2. **録画モード**
   - アナライザーの描画を録画し、動画ファイルとして出力する

本アプリは **HTML / CSS / JavaScript を中心に構成し、特別なライブラリのインストールなしで簡単に起動できること** を最重要要件とする。

## 2. 開発方針

### 2.1 基本方針
- ブラウザのみで動作する構成を優先する
- 可能な限り **外部ライブラリなし** で実装する
- ユーザーに `npm install` 等を要求しない構成を目指す
- 実装の中核には Web 標準 API を利用する
  - 音声解析: `Web Audio API`
  - 描画: `Canvas 2D API`
  - 録画: `MediaRecorder API`
  - 再生: `HTMLAudioElement` / `HTMLVideoElement`

### 2.2 UI方針
- UI は簡素でよいが、**視認性** と **直感的操作性** を重視する
- ビジュアライザー表示領域を主役とする
- 背景は常に黒固定とする
- コントロール部分は過度な装飾を避け、明快な構成にする

### 2.3 対象環境
- デスクトップブラウザを主対象とする
- 主な対象ブラウザ
  - Google Chrome
  - Microsoft Edge
- 初期段階ではモバイル最適化は優先度を下げる

## 3. 想定ユースケース

### 3.1 再生しながら鑑賞する
- ユーザーが音声ファイルまたは動画ファイルを読み込む
- 再生に合わせてビジュアライザーがリアルタイム表示される
- 再生中に色や動き、レイヤー構成を操作できる

### 3.2 複数ファイルを順番に流す
- 3つのファイルスロットにメディアを設定する
- 自動的に順番再生する
- 3番目の再生終了後は1番目に戻る

### 3.3 録画して書き出す
- 単一ファイルでアナライザーを再生する
- ビジュアライザー描画領域を録画する
- 録画結果を動画ファイルとして保存する

## 5. 用語定義

### 5.1 アナライザー
音声信号を解析して可視化する描画表現全般を指す。

### 5.2 レイヤー
音域ごと、または描画設定ごとに分けられた独立表示単位を指す。

### 5.3 0dB地点
可視化上の基準位置・基準辺・中心点を指す。  
厳密な音響計測値ではなく、表示上の「中立位置」の意味で用いる。

### 5.4 スロット
ユーザーがファイルを読み込むための枠。  
再生モードでは3つ存在する。

## 6. 画面構成

### 6.1 メイン画面構成

1. **ビジュアライザー表示領域**
2. **モード切替・再生制御領域**
3. **ファイルスロット領域**

5. **レイヤー設定領域**

- ビジュアライザー表示領域を最も大きく確保する
- 設定パネルは右側または下部に配置する
- 初版はデスクトップ前提の2カラムまたは上下分割構成を推奨する

## 7. ファイル入力仕様

### 7.1 入力対象
- 音声ファイル
- 動画ファイル

### 7.2 対応形式
ブラウザが標準的に再生可能な形式を基本対応とする。

#### 音声
- `.mp3`
- `.wav`
- `.ogg`
- `.m4a`
- `.aac`  
※ 一部はブラウザ依存

#### 動画
- `.mp4`
- `.webm`
- `.ogv`
- `.ogg`  
※ `.mov` などはブラウザ依存のため保証対象外

### 7.3 入力方法
- ファイル選択ダイアログ
- 将来的にドラッグ&ドロップを拡張可能とする

### 7.4 バリデーション
- 非対応形式の場合はエラーメッセージを表示する
- 読み込み失敗時は当該スロットにエラー状態を表示する

## 8. スロット・再生キュー仕様

### 8.1 スロット数
- 再生モードでは **3スロット**
- 録画モードでは **1スロットのみ有効**

### 8.2 再生順
- スロット1 → スロット2 → スロット3 → スロット1 → ...
- ループ循環を基本動作とする

### 8.3 スロット未設定時の挙動
- 次スロットが未設定なら、その次の設定済みスロットを探す
- すべて未設定なら再生停止する

### 8.4 スロットUI表示項目
各スロットには以下を表示する。
- スロット番号
- ファイル名
- ファイル種別（音声 / 動画）
- 読み込み状態
- 再生対象状態
- 選択ボタン
- 削除ボタン

### 8.5 操作
- 再生
- 一時停止
- 停止
- 次へ
- 前へ
- 対象スロットの手動選択

---

## 9. 音声解析仕様

### 9.1 解析方式
- `Web Audio API` を利用する
- `AudioContext`
- `MediaElementAudioSourceNode`
- `AnalyserNode`

### 9.2 取得データ
- 周波数スペクトラムデータ
- 必要に応じて時系列波形データ

### 9.3 更新方式
- `requestAnimationFrame` を使用して描画同期を行う

### 9.4 基本表現
- 原則として **全音域を1つのアナライザーで表現** する

### 9.5 拡張表現
- 音域を2〜4分割して複数レイヤー表示できる

## 10. レイヤー仕様

### 10.1 レイヤー数
選択可能なレイヤー数:
- 1
- 2
- 3
- 4

### 10.2 音域分割
初期実装では以下のような帯域分割を想定する。
- Low
- Low-Mid
- High-Mid
- High

### 10.3 各レイヤーの設定項目
- 表示 ON / OFF
- 対象帯域
- 色相
- 色相変化幅
- 輝度変化幅
- 彩度変化幅
- 感度
- 太さ / サイズ
- 透明度
- 表現タイプ

### 10.4 レイヤー合成
- 同一キャンバス上に重ね描きする
- 描画順はレイヤー番号順を基本とする
- 将来的にブレンドモード追加を可能にする設計とする

## 11. アナライザー表示仕様

### 11.1 背景
- 背景色は **常に黒固定**

### 11.2 アスペクト比
切替可能な表示比率:
- `16:9`
- `1:1`

### 11.3 表現タイプ
初期実装候補:
- 点
- 線
- 面
- 棒
- 円形 / 円環
- 放射
- ミラー対称

### 11.4 表現要素
- 点ベース表現
- 線ベース表現
- 面ベース表現
- 複合表現

### 11.5 0dB地点表現
以下の基準表現を切替可能とする。

- 長方形の底辺基準
- 長方形の中央基準
- 円形の中心基準
- 向かい合わせの四角の接辺基準
- 左右対称中心線基準
- 上下対称中心線基準

### 11.6 描画原則
- 再生音声に追従してリアルタイム更新
- 視認性を優先し、背景との十分なコントラストを保つ
- レイヤー数増加時にも破綻しない描画構造とする

## 13. 再生モード仕様

### 13.1 概要
- 3スロットキューを利用する通常鑑賞モード

### 13.2 動作
- 選択されたスロットのメディアを再生
- 音声解析を行いビジュアライザーを描画
- 終了時は次スロットへ自動遷移

### 13.3 動画ファイル利用時
- 動画の音声成分を解析対象とする
- 初版では動画映像のメイン表示は不要
- 必要なら小さなプレビュー表示は将来拡張とする

---

## 14. 録画モード仕様

### 14.1 概要
- 単一ファイルを再生しながらビジュアライザー表示を録画する

### 14.2 利用制約
- 利用可能スロットは1つのみ
- キュー再生は無効

### 14.3 録画対象
- ビジュアライザー描画領域のみを録画対象とする
- UIパネルは録画対象外とする

### 14.4 出力形式
- 初版の基本出力は `webm`
- `MediaRecorder` でサポートされる形式を優先する
- `mp4` 出力はブラウザ標準で安定しないため初版対象外とする

### 14.5 操作
- 録画開始
- 録画停止
- 保存
- 再録画

### 14.6 保存ファイル名
- 初期仕様では日時ベースの自動命名でよい
- 将来的に任意名称指定を検討する

### 15.3 操作性要件

## 16. 非機能要件

### 16.1 起動容易性
- クローン後すぐに利用開始しやすいこと
- できる限りビルド不要構成とする
- ローカルで静的ファイルとして扱える構成を優先する

### 16.2 パフォーマンス
- 一般的なPC環境で滑らかに動作すること
- 1〜4レイヤー構成でも極端に描画が破綻しないこと
- 録画時にも実用的なフレームレートを維持すること

### 16.3 保守性
- 音声処理、描画処理、UI制御、録画処理を分離する
- アナライザータイプ追加が容易な構造とする
- レイヤー機能の拡張がしやすい設計にする

### 16.4 拡張性
将来的に以下の拡張を想定可能な構造にする。
- プリセット保存 / 読込
- ブレンドモード
- 動画プレビュー表示
- 追加アナライザータイプ
- 録画品質設定
- 設定のURL共有やJSON保存

## 21. 受け入れ条件

以下を満たしたとき、初期仕様を達成したと判断する。

1. 音声ファイルまたは動画ファイルをブラウザで読み込める
2. 再生中にリアルタイムでアナライザーが表示される
3. アナライザータイプを複数から選択できる
4. 再生中に色相・彩度・輝度などを変更できる
5. 1〜4レイヤー構成を切り替えられる
6. 3スロットによる循環再生ができる
7. 録画モードで単一スロットの録画ができる
8. 録画結果を保存できる
9. 背景が常に黒である
10. 表示比率として 16:9 / 1:1 を選択できる
11. 特別なライブラリ導入なし、または最小限で起動できる

## 22. 制約・注意事項

- 対応メディア形式はブラウザ実装に依存する
- ブラウザの自動再生制限により、初回再生にはユーザー操作が必要
- `MediaRecorder` の出力形式はブラウザ差異がある
- `mp4` の安定出力は標準ブラウザ機能だけでは難しい
- 初版では高機能動画編集ではなく、**アナライザー録画** に目的を限定する
