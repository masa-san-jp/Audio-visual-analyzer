## 2. 各項目の設計

### 11.1 画質指定（§23「画質 / フレームレート指定録画」の残り半分）

FPS指定は実装済みのため、画質プリセットを追加して §23 の当該項目を完了させる。

- プリセット: **低 / 標準 / 高** = 0.08 / 0.15 / 0.25 bit/pixel/frame
  （現行の固定値 0.15 を「標準」とする）
- ビットレートの下限/上限（6〜24Mbps）もプリセット係数に比例してスケールする
  （低画質時に下限クリップで実質無効化されるのを防ぐ）
- 録画セクション・オフライン書き出しセクションそれぞれに画質セレクトを追加
  （FPSセレクトと同じ配置方針。両者は独立に選択できる）
- `js/recorder.js`: `setQuality(key)` を追加し `_videoBitrate()` に係数適用
- `js/offline-exporter.js`: `export()` の `opts.quality` として受け取り `_videoBitrate()` に係数適用
- 音声ビットレート（192kbps）は据え置き
- **DoD**: 低/標準/高で算出ビットレートが係数どおり変わり、書き出し・録画が完走する

### 11.2 再生シークバー・時間表示

- 再生セクションにシークバー（`<input type="range">`）と「現在位置 / 総時間」表示を追加
- `timeupdate` でバーと表示を更新。ドラッグ中は `timeupdate` による上書きを抑止
- バー操作で `mediaElement.currentTime` を変更（音声・動画共通）
- ファイル未読込・マイク入力中は無効化
- メディア要素はファイル読込ごとに作り直されるため、リスナーは読込成功時に毎回付け直す
- **DoD**: 任意位置へのシークが効き、時間表示が mm:ss 形式で追従する

### 11.3 音量調整

- 再生セクションに音量スライダー（0〜100%）を追加し `mediaElement.volume` に反映
- 選択値はセッション内で保持し、次のファイル読込時にも適用する（永続化はしない）
- マイク入力には適用しない（モニター音声の概念がないため）
- **DoD**: スライダーで再生音量が変わり、ファイルを切り替えても維持される

### 11.4 ドラッグ&ドロップ読込

- ビジュアライザー表示領域（`#visualizer-area`）へ音声/動画ファイルをドロップすると
  再生用ファイルとして読み込む（既存のファイル選択と同じ経路・同じ後処理）
- ドラッグ中は領域をハイライト表示する
- 非対応ファイル（`audio/`・`video/` 以外）は無視する
- 実装: ファイル選択の読込処理を `_loadMediaFile(file)` として抽出し、
  input change とドロップの両方から呼ぶ
- **DoD**: ドロップで読込→再生でき、マイク入力中のドロップはマイクを停止して切り替わる
