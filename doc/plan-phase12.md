## 1. スコープ（`doc/spec.md` §8 準拠）

- 再生モードで **3スロット**。各スロットに音声/動画ファイルを読込できる
- 再生順はスロット1 → 2 → 3 → 1 のループ循環。再生終了時に次の**設定済み**スロットへ
  自動で進んで再生を継続する（次が未設定ならさらに次を探す。全滅なら停止）
- スロットUI表示: スロット番号 / ファイル名 / 種別（音声・動画）/ 読み込み状態 /
  再生対象（アクティブ）状態 / 選択ボタン / 削除ボタン（§8.4）
- 操作: 再生 / 一時停止 / 停止 / **次へ / 前へ**（§8.5）
- 録画モードでは自動循環しない（現行どおり再生終了で録画停止。§8.1「1スロットのみ有効」の解釈）
- 録画中はスロット操作（読込/選択/削除/次へ/前へ）を受け付けない

## 2. 設計

### 2.1 `js/audio-engine.js`
- `createMediaElementSource()` は同一要素へ2回呼ぶと例外になるため、
  要素→ソースノードを `WeakMap` でキャッシュし、スロット再選択時に再利用する

### 2.2 `js/media-manager.js`
- 単一 `mediaElement` 管理から3スロット管理（`slots[3]` + `activeIndex`）へ拡張
- `loadFile(file, slotIndex)`: 指定スロットへ読込（省略時はアクティブスロット。既存互換）
- `selectSlot(index)`: アクティブ切替（旧要素を一時停止し、新要素を解析グラフへ接続）
- `clearSlot(index)`: スロットの破棄（objectURL解放を含む）
- `advance(dir)`: ±方向に次の設定済みスロットを循環探索して選択
- 既存の `mediaElement` / `isLoaded` / `isPlaying` はアクティブスロットを指す getter として維持
  （既存呼び出し箇所を壊さない）

### 2.3 `js/ui-controller.js`
- ファイルセクションに3スロット行（読込/選択/削除 + 名前/種別/状態表示）を追加。
  既存の `ファイルを選択` ボタン・ドラッグ&ドロップはアクティブスロットへの読込として維持
- 再生セクションに `次へ` / `前へ` ボタンを追加
- スロット切替時の共通処理（動画合成対象・シークバー/音量リスナー・ファイル名表示の付け替え）を
  `_activateSlot()` に集約
- `_onEnded()`: 再生モードでは次の設定済みスロットへ自動循環（1スロットのみならループ再生）。
  録画モードは現行どおり停止
