# 開発基本方針

## 仕様

- 仕様は `doc/spec.md` に従う
- 仕様の変更・追記が生じた場合は、指示に応じて `doc/spec.md` を書き換える

## 開発ログ

- 開発作業の記録は `log.md` に残す
- `log.md` は指示がなくても常に更新する
- 記録対象
  - 実装した機能・変更内容
  - フェーズの進捗
  - `doc/spec.md` を変更した場合はその内容と理由

## 開発方針

- まず最低限動くプロトタイプを作る（Phase 1 相当）
- その後フェーズを分けて機能を充実させる
- 各フェーズの区切りは `log.md` に記録する
- 不要なライブラリ導入・ビルド工程の追加は避ける

## ログフォーマット

```md
## YYYY-MM-DD

### 作業内容
- ...

### spec.md 変更（あれば）
- 変更箇所と理由

### 備考
- ...
```
